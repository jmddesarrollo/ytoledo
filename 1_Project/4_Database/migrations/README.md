# Database Migrations

This directory contains database migration scripts for the Y-Toledo project.

## Migration Files

### 001_add_file_fields_to_routes.sql
- **Date**: 2026-01-23
- **Description**: Adds file tracking fields to the routes table to support file attachments
- **Changes**:
  - Adds `file_track` VARCHAR(255) field with default empty string
  - Adds `filename_track` VARCHAR(255) field with default empty string
  - Updates existing records to ensure empty strings for new fields

## How to Apply Migrations

1. Connect to your MySQL database
2. Run the migration script:
   ```sql
   source /path/to/001_add_file_fields_to_routes.sql
   ```

## Migration Requirements

These migrations support the file attachment functionality as specified in:
- Requirements 6.1: Add file_track field
- Requirements 6.2: Add filename_track field  
- Requirements 6.3: Default empty values for file_track
- Requirements 6.4: Default empty values for filename_track

## Field Descriptions

- **file_track**: Unique identifier for the attached file, generated by the File_Manager.generateIdentifier() method
- **filename_track**: Original filename with extension, used for download operations

Both fields default to empty strings when no file is attached to a route.
